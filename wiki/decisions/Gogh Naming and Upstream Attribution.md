Gogh Naming and Upstream Attribution records that Gogh is the product name while all upstream project and skill names remain owned, cited, and unrebranded.

## What it is
- Decision: the product name is Gogh.
- Decision: the prior working name Taste Skill Brain is retired as the product label.
- Decision: the root repository folder renames to `~/Desktop/Gogh` at closeout.
- Decision: upstream names remain upstream names in prose, links, file labels, source notes, and command examples.
- Decision: `tasteskill.dev` remains Leon Lin's upstream site name.
- Decision: `Leonxlnx/taste-skill` remains Leon Lin's upstream repository name.
- Decision: `design-taste-frontend` remains the upstream install skill name.
- Decision: Taste Skill remains Taste Skill, not Gogh Skill.
- Decision: make-interfaces-feel-better remains Jakub Krehel's skill and article title.
- Decision: Impeccable remains Paul Bakaus's project and toolchain name.
- Decision: Anthropic frontend-design remains Anthropic's skill title.
- Decision: UI UX Pro Max remains nextlevelbuilder's skill name.
- Decision: Vercel web-design-guidelines remains Vercel's skill and guideline name.
- Decision: source notes keep upstream attribution visible instead of absorbing it into the Gogh brand.
- Decision: the stack layer may say Gogh when it describes selection, sequencing, conflict resolution, graph hygiene, and advisory outputs.
- Decision: the stack layer must name the upstream skill when it describes an upstream rule, command, install path, release, star count, license, or author.

## How it works
- Use Gogh for the brain, vault, stack advisor, root index, overview, release package, and operator-facing product wrapper.
- Use Taste Skill for Leon Lin's v2 project, dials, Section 14 gate, anti-slop bans, install command, official site, and repository.
- Use make-interfaces-feel-better or MIFB for Jakub Krehel's micro-interaction execution guidance.
- Use Impeccable for Paul Bakaus's command vocabulary, PRODUCT.md and DESIGN.md contracts, hooks, live mode, and detector rules.
- Use Anthropic frontend-design for the official baseline aesthetic-direction skill from Anthropic.
- Use UI UX Pro Max for the retrieval database skill, `uipro` CLI surface, style database, palettes, and UX rule references.
- Use Vercel web-design-guidelines for the runtime-fetched audit wrapper and Web Interface Guidelines findings.
- Keep source URLs and retrieved dates attached to current counts, versions, stars, releases, install counts, and licenses.
- Keep aliases narrow when a wikilink needs a reader-friendly label.
- Do not use aliases to make an upstream source note look like a different project.
- Do not replace an upstream author name with Gogh when the claim is about authorship, ownership, or release behavior.
- Do not make the repo folder rename part of upstream attribution.
- Treat the folder rename as local closeout housekeeping only.
- Keep old titles alive only when they are needed as historical context or search terms.
- Point new root navigation at Gogh canonicals, not the older single-skill framing.

## Pitfalls
- Rebranding `tasteskill.dev` as a Gogh site would misattribute Leon Lin's project.
- Rebranding `Leonxlnx/taste-skill` would hide the upstream repository owner.
- Rebranding `design-taste-frontend` would break the documented install path.
- Rebranding skill titles would make source-ledger rows harder to audit.
- Rebranding author names would turn provenance into marketing copy.
- Treating Gogh as the author of Impeccable detector findings would violate the product boundary.
- Treating Gogh as the source of Vercel file-line audit rules would hide the runtime-fetch mechanism.
- Treating UI UX Pro Max database counts as Gogh-owned facts would blur source responsibility.
- Treating Anthropic frontend-design as a Gogh invention would erase the baseline skill history.
- Treating MIFB values as Gogh-owned thresholds would hide Jakub Krehel's upstream craft layer.
- Renaming the repository folder before closeout could break active local paths during the integration slice.
- Leaving old single-skill root copy in place would conflict with the product rename.
- Using broad aliases could make link integrity pass while semantic attribution drifts.
- Moving raw captures would violate the immutable source-material rule.
- Updating ledgers from this slice would exceed the assigned ownership boundary.

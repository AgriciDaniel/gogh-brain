# Flows Hub

Flows turns the stack into ordered operator procedures for selection, build, audit, refresh, and intake.

## Catalog

### gogh/taste-skill

- [[Audit-First Redesign (Prompt 2)]] - Audit-First Redesign is the official Taste Skill flow for changing an existing site only after preservation and SEO constraints are explicit.
- [[Dial Tuning Guide]] - Dial Tuning Guide turns brief signals into DESIGN_VARIANCE, MOTION_INTENSITY, and VISUAL_DENSITY values.
- [[Greenfield Build (Prompt 1)]] - Greenfield Build is the official Taste Skill prompt flow for creating a new landing, portfolio, or marketing page from a brief.
- [[Image-First Reference Pipeline]] - Image-First Reference Pipeline is the Taste Skill workflow for resolving visual direction in images before implementation.
- [[Install and Load]] - Install and Load is the Taste Skill flow for adding design-taste-frontend and loading it once at the top of an agent session.
- [[Modernization Modes]] - Modernization Modes define whether Taste Skill should preserve, overhaul, or treat a redesign as greenfield-with-content-preserved.

### gogh/impeccable

- [[Impeccable Project Lifecycle]] - The Impeccable lifecycle turns setup, shaping, critique, refinement, hardening, live iteration, and detector gates into one command vocabulary.

### gogh/ui-ux-pro-max

- [[Design System Generation Flow]] - Design System Generation Flow turns a short product brief into a retrieved UI system with pattern, style, palette, typography, effects, anti-patterns, and checks.

### gogh/stack

- [[Audit Pipeline Flow]] - Audit Pipeline Flow orders the stack checks so deterministic gates run before advisory craft review.
- [[Full Stack Build Flow]] - Full Stack Build Flow is the flagship greenfield ordering: set direction, execute feel, persist contracts, detect slop, and audit the result.
- [[Skill Stack Selection Flow]] - Skill Stack Selection Flow is the operator decision tree that mirrors the local `scripts/render_stack_advisor.py` rules.

### gogh/ops

- [[Brain Refresh Flow]] - Brain Refresh Flow keeps Gogh current by following refresh_due, re-verifying fast-moving counts, and demoting stale claims instead of silently preserving them.
- [[Claim Verification Flow]] - Claim Verification Flow is the adversarial second-sourcing procedure that keeps Gogh from turning stale or misattributed claims into facts.
- [[New Skill Intake Flow]] - New Skill Intake Flow is the repeatable procedure for turning a design skill capture into source-cited Gogh notes, comparisons, and registry coverage.
